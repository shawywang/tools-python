# 八字四柱计算（公历输入）

## 概述

输入公历出生日期 + 时辰，直接输出四柱干支（年柱、月柱、日柱、时柱）。  
纯 Python 实现，仅依赖 `csv`、`datetime` 标准库和 `日柱表.csv`，**无 `lunardate` 依赖**。

### 文件结构

```
算八字/
├── bazi.py                 # 主程序：BaZi(year, month, day, hour, minute)
└── 日柱表.csv              # 日柱查表（1900-01-01 ~ 2100-12-31）
```

### 构造函数（三种调用方式）

```python
from bazi import BaZi

# 方式1：精确时间（时+分）
bz = BaZi(2026, 3, 7, 12, 0)

# 方式2：时辰名（只知地支不知具体几点）
bz = BaZi(1950, 6, 12, shichen="辰")  # → 8:00

# 方式3：只传日期（默认子时 0:00）
bz = BaZi(2026, 3, 7)
```

### 输出属性

| 属性 | 类型 | 说明 |
|---|---|---|
| `compact` | str | `"丙午 辛卯 庚辰 壬午"` |
| `four_pillars` | dict | `{"年柱":("丙","午"), "月柱":(...), ...}` |
| `year_gan`, `year_zhi` | str | 年干、年支 |
| `month_gan`, `month_zhi` | str | 月干、月支 |
| `day_gan`, `day_zhi` | str | 日干、日支 |
| `hour_gan`, `hour_zhi` | str | 时干、时支 |

```python

    # 1) 查 _TERM_CACHE → 有则复用
    # 2) 无 → stdin 交互输入 → 存入缓存
    # 3) 出生 datetime vs 节气 datetime 比较 → 返回结果
```

- `ret_year=True` → 返回年份（用于立春/年柱）
- `ret_year=False` → 返回地支字符串（用于月支）

### 运行时缓存：`_TERM_CACHE`

```python
_TERM_CACHE: dict = {}  # {year: {节名称: (month, day, hour, minute)}}
```

- 模块级字典，同一进程内跨 BaZi 实例共享
- 年柱问过立春后，月柱自动复用
- **同一年多次计算不会重复提问**

### JIE 常量（硬编码集合）

```python
JIE = [
    {1.4, 1.5, 1.6},    # 小寒 → 丑
    {2.3, 2.4, 2.5},    # 立春 → 寅
    {3.5, 3.6},         # 惊蛰 → 卯
    ...
    {12.6, 12.7, 12.8}, # 大雪 → 子
]
JIE_MIN = [min(s) for s in JIE]  # 预计算最早日期
```

- **不在模糊范围** → 用 `min(set)` 自动判断（无提示）
- **在集合中** → `key in days` 检测到 → 交互式弹提示
- 用 `month.day` 浮点数简洁直观，因为所有日期都是个位数（max 9）

## 四柱算法

### 年柱（以立春为界）

```
月份判断  1月及之前／2月3日前  → 前一年（自动）
          2月6日之后            → 当年（自动）
          2月3~5日（模糊）      → 弹提示问立春精确时间
```

年干: `(实际年份 - 4) % 10`  
年支: `(实际年份 - 4) % 12`

### 月柱（以12节为界）

月支由出生日期落在哪个节气区间决定。12个"节"（月支分界点）：

| 节 | 日期范围 | → 月支 |
|---|---|---|
| 小寒 | 1/4~6 | 丑 |
| 立春 | 2/3~5 | 寅 |
| ... | ... | ... |
| 大雪 | 12/6~8 | 子 |

- **不在任何节的日期范围** → 用 `JIE_MIN`（最早日期）自动判断
- **在某个节的日期范围内** → 弹提示问该节精确时间

月干：五虎遁 `(年干→寅月天干 + 月支偏移) % 10`

```text
五虎遁口诀：
  甲己之年丙作首 → 甲/己年 寅月天干=丙
  乙庚之年戊为头 → 乙/庚年 寅月天干=戊
  丙辛之岁寻庚上 → 丙/辛年 寅月天干=庚
  丁壬壬寅顺水流 → 丁/壬年 寅月天干=壬
  戊癸何方发，甲寅之上好追求 → 戊/癸年 寅月天干=甲
```

### 日柱（查表 + 公式回退）

1. 优先查 `日柱表.csv`（`YYYY-MM-DD` → 干支）
2. 查不到则用公式：以 2000-01-01（甲子日，天干索引=4，地支索引=6）为基准计算偏移

`_lookup_day_ganzhi(dt)` 是独立可复用的静态方法，支持任意 `datetime.date` 查询，子时边界变体也用它查次日日柱。

### 时柱（五鼠遁）

时辰地支由小时分钟决定，`_get_hour_zhi(total_min) → int`：

| 时辰 | 时间范围 | 地支索引 |
|---|---|---|
| 子 | 23:00~00:59 | 0 |
| 丑 | 01:00~02:59 | 1 |
| ... | ... | ... |
| 亥 | 21:00~22:59 | 11 |

时干：五鼠遁 `(日干→子时天干 + 时辰地支索引) % 10`

```text
五鼠遁口诀：
  甲己还加甲 → 甲/己日 子时天干=甲
  乙庚丙作初 → 乙/庚日 子时天干=丙
  丙辛从戊起 → 丙/辛日 子时天干=戊
  丁壬庚子居 → 丁/壬日 子时天干=庚
  戊癸何方发，壬子是真途 → 戊/癸日 子时天干=壬
```

## 子时边界特殊处理（23:00~23:59）

当出生时间落在 23:00~23:59（子时的前半段），`__init__` 自动调用 `_compute_zishi_variants()` 输出 **3 种不同算法结果**：

| # | 规则 | 日柱 | 时柱 |
|---|------|------|------|
| 1 | 23~24 算次日，日时都变 | 次日干支（查表） | 五鼠遁基于次日日干，子时 |
| 2 | **23~24 同当日 0~1，日时不变（通常建议）** | 当日 | 五鼠遁基于当日日干，子时 |
| 3 | 亥时天干 +1 作为子时天干，只变时柱 | 当日 | `TIAN_GAN[(亥时_gan + 1) % 10]` + 子 |

输出示例：
```text
⚠️ 出生时间 23:45 处于子时(23~1点)的23~24交界，特殊处理3个八字：
  第一种，23~24算次日，日时都变：  癸酉 辛酉 丁酉 庚子
  第二种（通常建议），23~24同当日0~1，日时都不变：癸酉 辛酉 丙申 戊子
  第三种，9~11的时干，再往后数一个天干作为时干，只变时柱：癸酉 辛酉 丙申 庚子
```

`compact` / `four_pillars` 默认返回第二种（当日不变）。

## 交互流程

程序运行时会自动判断，仅在以下情况弹提示询问用户：

1. 出生日期在 **立春前后**（2月3~5日）→ 问立春精确时间（用于年柱）
2. 出生日期在 **某个月节前后**（如上表中日期范围）→ 问该节精确时间（用于月柱）

查询参考网站：https://dijizhou.100xgj.com/jieqibiao/

输入格式：`月 日 时 分`（空格分隔，24小时制），如 `2 4 4 1`。

同一程序运行期间会缓存已输入的节气时间，不会重复询问。
