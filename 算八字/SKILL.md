# 八字四柱提取（公历版）

## 概述

输入公历出生日期 + 时辰，直接输出四柱干支（年柱、月柱、日柱、时柱）。  
纯 Python 实现，仅依赖 `csv`、`datetime` 标准库和 `日柱表.csv`。

### 文件结构

```
算八字/
├── bazi_extract_lunar.py   # 主程序
└── 日柱表.csv              # 日柱查表（1900-01-01 ~ 2100-12-31）
```

---

## 快速使用

```python
from bazi_extract_lunar import BaZi

# 方式1：精确时间（时+分）
bz = BaZi(2026, 3, 7, 12, 0)
print(bz.compact)          # → 丙午 辛卯 庚辰 壬午

# 方式2：时辰名（只知地支不知具体几点）
bz = BaZi(1950, 6, 12, shichen="辰")
print(bz.compact)          # → 庚寅 壬午 戊寅 丙辰

# 方式3：只传日期（默认子时0:00）
bz = BaZi(2026, 3, 7)
print(bz.compact)          # → 丙午 辛卯 庚辰 丙子
```

### 输出属性

| 属性 | 类型 | 说明 |
|---|---|---|
| `compact` | str | `"丙午 辛卯 庚辰 壬午"` |
| `four_pillars` | dict | `{"年柱":("丙","午"), "月柱":(...), ...}` |
| `year_gan`, `year_zhi` | str | 年干、年支 |
| `month_gan`, `month_zhi` | str | 月干、月支 |
| `day_gan`, `day_zhi` | str | 日干、日支 |
| `hour_gan`, `hour_zhi` | str | 时干、时支 |

## 四柱算法

### 年柱（以立春为界）

```
          1月及之前         → 前一年
月份判断  2月3日之前        → 前一年
          2月6日之后        → 当年
          2月3~5日（模糊）  → 弹提示问用户
```

年干: `(实际年份 - 4) % 10`  
年支: `(实际年份 - 4) % 12`

### 月柱（以12节为界）

月支由出生日期落在哪个节气区间决定。12个"节"（月支分界点）：

| 节 | 日期范围 | → 月支 |
|---|---|---|
| 小寒 | 1/4~6 | 丑 |
| 立春 | 2/3~5 | 寅 |
| 惊蛰 | 3/5~6 | 卯 |
| 清明 | 4/4~6 | 辰 |
| 立夏 | 5/5~7 | 巳 |
| 芒种 | 6/5~7 | 午 |
| 小暑 | 7/6~8 | 未 |
| 立秋 | 8/7~9 | 申 |
| 白露 | 9/7~9 | 酉 |
| 寒露 | 10/7~9 | 戌 |
| 立冬 | 11/7~9 | 亥 |
| 大雪 | 12/6~8 | 子 |

- **不在任何节的日期范围** → 用各节最早日期自动判断
- **在某个节的日期范围内**（如上表）→ 弹提示问用户该节精确时间

月干：五虎遁 `(年干→寅月天干 + 月支偏移) % 10`

```
五虎遁口诀：
  甲己之年丙作首 → 甲/己年 寅月天干=丙
  乙庚之年戊为头 → 乙/庚年 寅月天干=戊
  丙辛之岁寻庚上 → 丙/辛年 寅月天干=庚
  丁壬壬寅顺水流 → 丁/壬年 寅月天干=壬
  戊癸何方发，甲寅之上好追求 → 戊/癸年 寅月天干=甲
```

### 日柱（查表 + 公式回退）

1. 优先查 `日柱表.csv`（`YYYY-MM-DD` → 干支）
2. 查不到则用公式：以 2000-01-01（甲子日）为基准计算偏移天数

### 时柱（五鼠遁）

时辰地支由小时分钟决定：

| 时辰 | 时间范围 | 地支索引 |
|---|---|---|
| 子 | 23:00~00:59 | 0 |
| 丑 | 01:00~02:59 | 1 |
| 寅 | 03:00~04:59 | 2 |
| 卯 | 05:00~06:59 | 3 |
| 辰 | 07:00~08:59 | 4 |
| 巳 | 09:00~10:59 | 5 |
| 午 | 11:00~12:59 | 6 |
| 未 | 13:00~14:59 | 7 |
| 申 | 15:00~16:59 | 8 |
| 酉 | 17:00~18:59 | 9 |
| 戌 | 19:00~20:59 | 10 |
| 亥 | 21:00~22:59 | 11 |

时干：五鼠遁 `(日干→子时天干 + 时辰地支索引) % 10`

```
五鼠遁口诀：
  甲己还加甲 → 甲/己日 子时天干=甲
  乙庚丙作初 → 乙/庚日 子时天干=丙
  丙辛从戊起 → 丙/辛日 子时天干=戊
  丁壬庚子居 → 丁/壬日 子时天干=庚
  戊癸何方发，壬子是真途 → 戊/癸日 子时天干=壬
```

## 子时边界特殊处理（23:00~23:59）

当出生时间落在 23:00~23:59（子时的前半段），输出 3 种不同算法结果：

```
⚠️ 出生时间 23:45 处于子时(23~1点)的23~24交界，特殊处理3个八字：
  第一种，23~24算次日，日时都变：  癸酉 辛酉 丁酉 庚子
  第二种（通常建议），23~24同当日0~1，日时都不变：癸酉 辛酉 丙申 戊子
  第三种，9~11的时干，再往后数一个天干作为时干，只变时柱：癸酉 辛酉 丙申 庚子
```

`compact` / `four_pillars` 默认返回第二种（当日不变）。

## 交互流程

程序运行时会自动判断，仅在以下情况弹提示询问用户：

1. 出生日期在 **立春前后**（2月3~5日）→ 问立春精确时间（用于年柱）
2. 出生日期在 **某个月节前后**（如3月5~6日近惊蛰）→ 问该节精确时间（用于月柱）

查询参考网站：https://dijizhou.100xgj.com/jieqibiao/

输入格式：`月 日 时 分`（空格分隔，24小时制），如 `2 4 4 1`。

同一程序运行期间会缓存已输入的节气时间，不会重复询问。

## 生辰.json 输入格式

```json
{
  "姓名": [
    {"生": "农历描述、公历年月日+时辰"},
    {"八字": "庚午年，己卯月，丙子日，庚寅时"}
  ]
}
```

支持的"生"字段公历部分格式：
- `1930年3月27寅时` → 日期 + 时辰名
- `1993年9月12日23:45` → 日期 + 精确时间
- `2010年7月9日7:10` → 含"日"字 + 精确时间

---

## 关键常量说明

| 常量 | 说明 |
|---|---|
| `JIE` | 12个月节的日期集合（`{month.day}` 浮点数格式） |
| `JIE_MING` | 12节名称列表 |
| `JIE_ZHI` | 12节对应的地支列表 |
| `JIE_MIN` | 预计算各节最早日期 |
| `SHICHEN_MAP` | 时辰名→具体时间映射（取中点） |
| `WU_HU_DUN` | 五虎遁字典：年干→寅月天干索引 |
| `WU_SHU_DUN` | 五鼠遁字典：日干→子时天干索引 |
| `_TERM_CACHE` | 运行时缓存用户输入的节气时间 |
| `_DAY_LOOKUP` | 日柱表CSV缓存 |

---

## 日柱表格式

`日柱表.csv`：`YYYY-MM-DD,天干,地支`，每行一日，覆盖 1900~2100年。
